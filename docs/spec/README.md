## プロジェクト概要

SNS Battle は、ライブ配信・1対1ビデオ通話マッチング・バトル形式の対戦を組み合わせたリアルタイムソーシャルプラットフォームである。

**3つのメイン機能**:

1. **配信機能**: 1対多のライブ配信。コメント・スタンプによるインタラクション
2. **マッチング機能**: 1対1のビデオ通話。トークテーマ表示・リアクション共有による楽しいマッチング体験
3. **SNS Battle 機能**: 1対1の対戦を観客付きで実施。スタンプ対決・ターン制トーク

初期リリースは Web 版のみ。将来的に Mobile（Expo）対応予定。

### デザイン原則

- **ダークファースト**: 全画面ダークモードベース。ライトモードは初期リリースでは非対応
- **グラデーションファースト**: CTA・見出し・インジケーターは `linear-gradient(135deg, #CBACF9 0%, #0EA5E9 50%, #CBACF9 100%)` などの3点グラデで構成。`bg-clip-text` でテキスト自体にグラデを適用するパターンを多用
- **グラスモーフィズム**: カード、トークテーマ表示、コントロールバーは半透明 + `backdrop-filter: blur(...)` を基本。境界には 1px の白透過枠を必ず付与
- **動的グロー演出**: スポットライト中のビデオ、ボタンホバー、選択中の選択肢などは `box-shadow` で内外グローを重ねる
- **アニメーション**: ページ遷移・状態遷移・要素出現は Framer Motion で滑らかに実装。spring 物理ベースを基本とし、`easeInOut` を併用
- **背景装飾**: 各ページに大きな blur オーブ（`blur(120-150px)`、低不透明度のパープル/シアン）を `pointer-events-none fixed` で配置し、奥行きを演出
- **コンパクトサイドバー**: 折りたたみ可能（240px / 68px）。アクティブ項目はパープル枠 + パープルグロー背景
- **ライブ感の演出**: LIVE バッジは赤グラデ背景 + 点滅ドット。視聴者数・スタンプ数はリアルタイム更新
- **バトル感の演出**: VS 画面は左右分割。中央に円形 VS バッジ。スタンプカウントは左右からせり出す対立プログレスバー（パープル vs ピンク）

## 機能一覧

### 認証機能

- Google OAuth でのログイン / サインアップ
- JWT トークンによるセッション管理（既存実装を活用）
- 将来対応: TikTok, X(Twitter), Instagram OAuth
- ユーザープロフィール（アバター、表示名、自己紹介）

### 配信機能（Live Streaming）

| 機能 | 詳細 |
|------|------|
| 配信開始 | ブラウザカメラ/マイクから直接配信（WHIP）。将来的に OBS 対応（RTMP） |
| カウントダウン | 配信開始前に 3, 2, 1 のカウントダウンアニメーション表示 |
| 視聴 | 配信中のストリームを他ユーザーがリアルタイム視聴 |
| コメント | 視聴者がテキストコメントを送信。配信画面にリアルタイム表示 |
| スタンプ | 視聴者がスタンプ（絵文字/カスタム画像）を送信。配信者画面にアニメーション表示 |
| 視聴者数表示 | 現在の視聴者数をリアルタイム表示 |
| 配信終了 | 配信者が手動で終了。視聴者に終了通知 |

### マッチング機能（1対1ビデオ通話）

| 機能 | 詳細 |
|------|------|
| マッチング待機 | ユーザーがマッチングキューに参加。相手が見つかるまで待機画面表示 |
| マッチング成立 | 2人が揃ったらマッチング成立通知 |
| カウントダウン | ビデオ通話開始前に 3, 2, 1 のカウントダウン表示 |
| ビデオ通話 | 1対1のリアルタイムビデオ通話（カメラ + マイク） |
| 時間制限 | 合計 10 分で自動終了。5 分経過後に「終了ボタン」が有効化 |
| トークテーマ表示 | 1 分ごとにマスターデータから取得したトークテーマを両者の画面に表示。例）お互い１分間ずつ自己紹介してください。好きなお酒を答えてください。直近の恋愛話をしてくださいなどです |
| 選択肢・印象ボタン | トークテーマに対する回答を選択肢ボタンで表示。両者が選択 |
| リアクション一致 | 同じ回答を選んだ場合、紙吹雪エフェクトを表示 |
| 残り時間表示 | 通話の残り時間をプログレスバーで常時表示 |
| マッチング終了 | 時間切れまたは手動終了。終了後に感想画面（任意） |
| マッチング後のメッセージ機能| 将来的にはマッチング終了後にお互いにgoodだったら、チャットメッセージが可能になる|

### SNS Battle 機能

| 機能 | 詳細 |
|------|------|
| バトル作成 | ユーザーがバトルルームを作成し、対戦相手を待機 |
| バトル参加 | 対戦相手がバトルルームに参加 |
| 観戦参加 | 他ユーザーが観客としてバトルを視聴 |
| カウントダウン | バトル開始前に 3, 2, 1 のカウントダウン表示 |
| ターン制トーク | 1 分ずつ交互にトーク。現在のターンを画面に表示。マイクのミュート切替で制御 |
| バトルトークテーマ | バトル用のトークテーマを両者と観客の画面に表示 |
| スタンプ対決 | 観客がスタンプを送信。各プレイヤーへのスタンプ数をリアルタイム集計・表示 |
| 勝敗判定 | スタンプ数の多い方が勝利。結果画面を表示 |
| 観客コメント | 観客がコメントを送信。画面にリアルタイム表示 |

### 共通機能

| 機能 | 詳細 |
|------|------|
| カウントダウン | 3, 2, 1 + 「START!」のフルスクリーンオーバーレイアニメーション |
| スタンプシステム | 定義済みスタンプ一覧から選択して送信。画面上をフロートするアニメーション |
| ユーザープロフィール | アバター、表示名、自己紹介、年齢（生年月日から自動計算）、性別（男性/女性/その他）の表示・編集。将来: MBTI、居住地域 |
| 通知 | マッチング成立、バトル招待などのリアルタイム通知 |

## 将来構想

### 課金・ショップシステム

- **コイン**: アプリ内通貨。App Store / Google Play の IAP、または Stripe で購入
- **スタンプショップ**: カテゴリ別（GENERAL / BATTLE / MATCHING）にスタンプを閲覧・購入。プレビューアニメーション付き
- **エフェクトショップ**: マッチング・バトル中に使えるエフェクト（紙吹雪、花火、ハート等）を購入
- **所持品管理**: 購入済みスタンプ・エフェクトの一覧表示
- DB設計は `docs/spec/common/README.md` に定義済み（stamp_masters, effects, user_inventory, coin_transactions）

### MBTI・会話アシスト

- **MBTI 設定**: プロフィールで16タイプから選択（INTJ, ENFP 等）
- **相性表示**: マッチング時に相手との MBTI 相性スコアを表示
- **会話アシスト**: MBTI 相性に基づいてトークテーマを最適化、相手への質問提案を自動生成
- **マッチング優先**: 相性の良い相手を優先的にマッチング（オプション）

### スタンプ・エフェクト拡張

- **アニメーション種別**: FLOAT（浮遊）/ BOUNCE（跳ねる）/ EXPLODE（爆発）/ SHAKE（振動）
- **マッチング中のスタンプ**: ビデオ通話中に相手画面にスタンプを送信、アニメーション付きで表示
- **エフェクト発動**: 選択肢の回答一致時やフリートーク中にエフェクトを発動
- **限定スタンプ**: 期間限定・イベント限定のプレミアムスタンプ
